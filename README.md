# PulseRead 后台管理系统

> AI 驱动的跨域资讯聚合平台 - 后台管理系统

## 技术栈

| 技术 | 版本 | 说明 |
|---|---|---|
| Vue | 3.4.x | 前端框架 |
| TypeScript | 5.4.x | 类型系统 |
| Ant Design Vue | 4.2.6 | UI 组件库 |
| Pinia | 2.1.x | 状态管理 |
| pinia-plugin-persistedstate | 3.2.x | 状态持久化 |
| Vue Router | 4.3.x | 路由管理 |
| Vite | 5.1.x | 构建工具 |
| axios | 1.6.x | HTTP 客户端 |
| ECharts | 5.5.x | 数据可视化 |
| dayjs | 1.11.x | 日期处理 |

## 快速开始

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 构建生产版本
npm run build
```

## 功能模块

### 仪表板
- 总资讯数、活跃用户、待处理反馈、系统可用率统计卡片
- 近 7/30/90 天资讯发布趋势折线图（ECharts）
- 领域分布饼图（科技 45%、财经 28%、政策 15%、商情 8%）
- 最近活动列表（6条）
- 快捷操作入口（新增资讯、处理反馈、用户管理、降噪配置）
- 系统状态监控（API、数据库、AI 服务、CDN）

### 资讯管理
- 资讯列表（12条 Mock 数据，支持关键词/领域/地区/评分筛选）
- 新增/编辑资讯表单（标题、摘要、领域、地区、来源、AI评分、标签）
- 批量删除功能

### 用户管理
- 用户列表（6位 Mock 用户，支持套餐/状态筛选）
- 用户详情弹窗（订阅领域、雷达词、阅读量等）
- 封禁/解封操作

### 用户反馈
- 反馈列表（12条 Mock 数据，含 Bug/功能建议/内容问题分类）
- 回复功能
- 状态流转（待处理 → 处理中 → 已解决 → 已关闭）

### 系统设置
- **降噪配置**：质量阈值（高质量 80 / 中等 60 / 过滤 40）、过滤规则开关、去重参数、来源可信度管理
- **领域管理**：科技/财经/政策/商情四大领域配置
- **地区管理**：国内/国际地区配置
- **雷达词管理**：全局推荐雷达词维护

## 目录结构

```
src/
├── api/          # API 接口（含 Mock 实现）
├── assets/       # 全局样式
├── components/
│   └── layout/   # 管理后台布局
├── router/       # 路由配置（含权限守卫）
├── stores/       # Pinia 状态管理
├── types/        # TypeScript 类型定义
├── utils/        # Mock 数据 & 工具函数
└── views/        # 页面组件
    ├── dashboard/
    ├── articles/
    ├── users/
    ├── feedback/
    └── settings/
```

## 接入后端

1. 修改 `src/api/index.ts` 中各 API 方法，将 Mock 实现替换为 `request.get/post/put/delete` 调用
2. 在 `.env.local` 中配置 `VITE_API_BASE_URL` 为后端服务地址
3. 根据后端接口文档调整请求/响应数据结构
